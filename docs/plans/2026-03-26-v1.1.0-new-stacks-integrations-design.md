# v1.1.0 Design: New Stacks + Integrations

**Date:** 2026-03-26
**Status:** Approved

## Goal

Add 3 new backend stacks (Rust/Axum, .NET/ASP.NET, Elixir/Phoenix) and 4 new integration enhancers (Redis cache, message queues, WebSockets, S3 storage) to create-kickstart.

## New Backend Stacks

### Rust (Axum)

- **Scaffold:** Template-based (`src/templates/axum/`)
- **Files:** `Cargo.toml`, `src/main.rs` (Axum router, health endpoint, JSON responses)
- **Language key:** `rust`
- **Port:** 8080
- **Why template:** No official Axum starter CLI exists

### ASP.NET Minimal API

- **Scaffold:** CLI-based — `dotnet new webapi --name {{name}} --no-https`
- **Language key:** `csharp`
- **Port:** 5000
- **Why CLI:** `dotnet new` is the standard .NET scaffolder

### Elixir (Phoenix)

- **Scaffold:** CLI-based — `mix phx.new {{name}} --no-html --no-assets --no-mailer`
- **Language key:** `elixir`
- **Port:** 4000
- **Why CLI:** `mix phx.new` is the standard Phoenix scaffolder

## Enhancer Support for New Languages

Every existing enhancer that is language-aware must handle the 3 new stacks:

| Enhancer | Rust (Axum) | C# (ASP.NET) | Elixir (Phoenix) |
|----------|-------------|---------------|------------------|
| docker | Multi-stage cargo build | dotnet publish | mix release |
| ci | cargo test + clippy | dotnet test + format | mix test + format |
| lint | clippy + rustfmt | dotnet format | mix format + credo |
| test | cargo test scaffold | xUnit test project | ExUnit tests |
| sample-crud | Axum handlers + serde | Minimal API endpoints | Phoenix controllers |
| logging | tracing crate | Serilog | Logger (built-in) |
| auth | jsonwebtoken crate | ASP.NET Identity + JWT | Guardian + JWT |
| doctor | check rustc, cargo | check dotnet | check elixir, mix |

## New Integration Enhancers

### `cache` — Redis

- Redis client setup per backend language (redis-py, ioredis, go-redis, redis-rs, StackExchange.Redis, Redix)
- Docker Compose: `redis:7-alpine` on port 6379
- `.env`: `REDIS_URL=redis://localhost:6379`
- Helper module: `cache.get/set/del` with TTL support
- Health check extended to include Redis connectivity

### `queue` — Message Queue

Interactive prompt asks: RabbitMQ or Kafka?

**RabbitMQ:**
- Docker Compose: `rabbitmq:3-management` (5672 + 15672 mgmt UI)
- Publisher + Consumer scaffold per language
- `.env`: `RABBITMQ_URL=amqp://guest:guest@localhost:5672`
- Example: `items.created` event on `POST /api/items`

**Kafka:**
- Docker Compose: KRaft-mode Kafka (no Zookeeper)
- Producer + Consumer scaffold per language
- `.env`: `KAFKA_BROKERS=localhost:9092`
- Example: `items.created` topic on `POST /api/items`

### `websocket` — Real-time Communication

- Backend: WebSocket endpoint at `/ws` per framework
  - FastAPI WebSocket, Django Channels, ws for Express, gorilla/websocket for Go, axum ws for Rust, ASP.NET WebSocket middleware, Phoenix Channels
- Frontend: `useWebSocket` hook (React), composable (Vue), store (Svelte)
- Example: live item updates — create in one client, appears in others
- No extra Docker service needed

### `storage` — S3/Object Storage

- Backend: `POST /api/upload` with presigned URL support per language (boto3, @aws-sdk/client-s3, aws-sdk-go-v2, aws-sdk-s3, AWSSDK.S3, ExAws.S3)
- Docker Compose: MinIO service (S3-compatible) on ports 9000 + 9001
- `.env`: `S3_BUCKET`, `S3_ENDPOINT`, `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`
- Frontend: drag-and-drop upload component
- `scripts/create-bucket.sh` for local dev setup

## Implementation Order

1. **Types + registry** — Add new stacks/enhancements to types.ts, registry.json, prompts.ts, cli.ts
2. **Axum template** — Create `src/templates/axum/` with Cargo.toml + main.rs
3. **ASP.NET + Phoenix registry** — CLI-based, just registry entries
4. **Update existing enhancers** — docker, ci, lint, test, sample-crud, logging, auth, doctor for 3 new langs
5. **New enhancer: cache** — Redis integration
6. **New enhancer: queue** — RabbitMQ/Kafka integration
7. **New enhancer: websocket** — Real-time scaffold
8. **New enhancer: storage** — S3/MinIO integration
9. **Tests** — Add tests for new stacks + enhancers
10. **Bump to v1.1.0** — Update package.json, changelog, publish
